## Task 1: Understand Infrastructure as Code
### 1. What is Infrastructure as Code (IaC)? Why does it matter in DevOps?
  -  Infrastructure as code means instead of creating infrastructure manually or by clicking buttons, you write code to create and manage it automatically just my on single click. We treat infrastructure(servers, databases, networks) exact same as like developer treat application code. Infrastructure code is just like blueprint. DevOps focus on automation, speed, reliability that all contains in the IAC.                                      
### 2. What problems does IaC solve compared to manually creating resources in the AWS console?
  - Terraform solves problems as:
    - Eleminated manual error
    - Automation at a scale
    - Modular and reusable infrastructure
    - Consistency across teams
    - Minimize human error
    - Declarative state management
### 3. How is Terraform different from AWS CloudFormation, Ansible, and Pulumi?
  - All the tools(terraform, aws cloudformation, ansible and pulumi) done the same work to automate the infrastructure but each one solves and used for different problems like:
  - terraform used for creating and managing cloud infrastructue.
  - AWS CloudFormation used only for infrastructure automation.
  - Ansible used for configuration the servers.
  - Pulumi used for IAC using programming languages.

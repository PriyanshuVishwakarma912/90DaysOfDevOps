## Task 1: Understanding Branches
  - 1.What is branch in Git?
      - A git branch is a separate workspace used to make changes without affecting the main project. Once the work is completed the changes can  be merged into the main branch.Branch manages different task independently.
  - 2. Why do we use branches instead of committing everything to main?
     - Because main represents the production ready line of development, commiting everything to main increases the chance of risk,instability and coordination problems at the time of production in the real world use case scenario.
  - 3. What is HEAD in Git?
     - HEAD is a special pointer in git that refers to the currently cheched-out commit.
  - 4. What happens to your files when you switch branches?
     - When switching branches git updates the working directory and staging area to match the snapshop of the commit that the target branch points to.
